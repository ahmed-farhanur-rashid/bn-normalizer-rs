# Third-Party Notices

This project (`bn-normalizer-rs`) is a Rust reimplementation of the
word-level Bangla Unicode normalization rules originally implemented in
[bnunicodenormalizer](https://github.com/mnansary/bnUnicodeNormalizer)
by Bengali.AI, in association with the OCR Team, APSIS Solutions
Limited, and BUET CSE NLP.

## What was derived from upstream

- The core word-level normalization rules and their execution order
  (`src/word/` in this repo)
- The Bangla language data tables: vowels, consonants, diacritics,
  conjuncts, complex roots, legacy symbol maps, nukta maps
- The correctness test oracle was generated by running the upstream
  Python library over sample text and capturing its output as ground
  truth

## What is original to this repo

- The sentence-level normalization module (`src/sentence/`), including
  tokenization, Bangla-script auto-detection, emoji/punctuation
  handling, and the `NoneTokenPolicy` configuration system — this
  functionality does not exist in the upstream library (see upstream
  issues #13 and #9, which confirm sentence-level handling is an open,
  unsolved problem upstream)
- The Rust implementation itself, PyO3 bindings, and packaging
